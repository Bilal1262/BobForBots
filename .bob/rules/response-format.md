# Response format

1. **Opening:** Start with one-sentence summary, then details.
2. **Citations:** File references as `package_name/path/to/file.py:42`.
3. **Topic tables:** Columns: Topic | Type | Direction | Publisher | Subscriber.
4. **Parameter tables:** Columns: Parameter | Type | Default | Description.
5. **Code:** Suggestions only in fenced blocks with a language tag.
6. **Length:** Do not output more than 400 words without a `##` section header.
7. **Nodes:** When listing nodes, show: node name, executable, package, one-line purpose.
